## 报告安全漏洞

如果你发现了安全漏洞，请**不要**在公开的 GitHub Issue 中报告。

请通过以下方式联系我们：

- 在 GitHub 仓库中使用 **Security > Report a vulnerability** 功能

我们会在 48 小时内回复你的报告，并在确认漏洞后尽快发布修复版本。

## 安全最佳实践

在使用本项目时，请注意以下安全事项：

1. **SECRET_KEY**：务必在 `.env` 中设置强随机密钥，不要使用默认值
2. **数据库**：生产环境请使用 PostgreSQL 并配置安全的连接字符串
3. **HTTPS**：生产环境请务必配置 HTTPS
4. **依赖更新**：定期更新依赖以获取安全补丁
